## A. `frontend/lib/api/**` ÔÇö the typed API client (8 files)

The client is split into: one transport core (`client.ts`), one type contract (`types.ts`),
five endpoint modules grouped by domain (`auth`, `runs`, `intake`, `alignment`, `awards`),
and one barrel (`index.ts`). WHY this shape: a single choke-point (`client.ts`) enforces the
cross-cutting rules ÔÇö `credentials:"include"`, typed `ApiError`, the 2FA branch, the
`gate_required` branch, `cache:"no-store"`, the API-base bake ÔÇö **once**, so no endpoint
module can forget them; the domain modules are thin, declarative `apiFetch<T>()` wrappers that
only name the method, path, and `T`. This is the storeÔåöview seam (ADR-0002): the frontend is a
**pure client** of the FastAPI backend; nothing here holds server state.

### A1. `lib/api/client.ts` ÔÇö transport core
- **path:** `/home/user/KR_RFP/frontend/lib/api/client.ts`
- **ext:** `.ts` ┬À **empty?** no ┬À **lines:** 251 ┬À **census:** row 317 (8086 bytes)
- **what:** The single typed `fetch` wrapper for the whole app. Exports the `ApiError` class,
  the `ApiErrorCode` type, and four transport functions: `apiFetch` (JSON), `apiUpload`
  (multipart), `apiDownload` (binary/blob), plus two internal helpers (`extractError`,
  `throwApiError`, `filenameFromDisposition`).
- **DETAILED WHY:** This file is where every cross-cutting HTTP rule is enforced exactly once
  so the rest of the client cannot drift. The session lives in an **httpOnly cookie**, so every
  request must send `credentials:"include"` ÔÇö if any endpoint module forgot that, the call would
  be silently unauthenticated. Centralizing it here makes that impossible. It also normalizes the
  backend's two error shapes (structured envelope vs FastAPI plain `{detail}`) into one typed
  `ApiError`, and lifts two special server signals into typed booleans the UI branches on (2FA
  required, gate required). Without this file every component would re-implement fetch, error
  parsing, and cookie handling, and they would disagree.

**Module constants (the API-base bake):**
- `API_BASE_URL = process.env.NEXT_PUBLIC_API_BASE_URL ?? "http://localhost:8000"` (line 9ÔÇô10).
  **WHY / the bake:** `NEXT_PUBLIC_*` vars are **inlined at build time** by Next.js ÔÇö this
  string is frozen into the client bundle at `next build`, **not** read at runtime. So the
  deployed backend URL must be passed as a `--build-arg` (see `Dockerfile` ARG, `deploy.sh`),
  and `??` gives a safe local default (`http://localhost:8000`) so a bare build still runs.
- `API_PREFIX = "/api/v1"` (line 12). Every path the domain modules pass is **relative to this
  prefix** ÔÇö they pass `/runs`, this prepends base+prefix ÔåÆ `ÔÇĞ/api/v1/runs`. Versioned API seam.
- `TWO_FACTOR_DETAIL = "2FA code required"` (line 15). The **exact** detail string the backend
  returns on a 401 to demand a TOTP code. Compared case-insensitively (see `ApiError`).

**`ApiError` class (exported, lines 26ÔÇô56):** extends `Error`.
- Fields: `status: number`, `detail: string`, `code: ApiErrorCode | null`,
  `twoFactorRequired: boolean` (all `readonly`).
- `constructor(status, detail, code = null)`: message = `detail || "Request failed with status {status}"`;
  sets `name = "ApiError"`. **`twoFactorRequired` derivation:** `status === 401 && detail.trim().toLowerCase() === "2fa code required"`.
  WHY case-insensitive + trim: defensive against server whitespace/casing variance on this one
  load-bearing string.
- `get isUnauthenticated()`: `(status === 401 || code === "unauthenticated") && !twoFactorRequired`.
  **WHY the exclusion:** the 2FA prompt is *also* a 401 but the user is **mid-login**, not
  signed-out ÔÇö the guard must not bounce them to the login-redirect path. This is the precise
  branch that keeps the TOTP step alive.
- `get isGateRequired()`: `code === "gate_required"`. **WHY:** the backend returns this code when
  a prerequisite step is incomplete (e.g. trying to freeze before analysis, or finalize before a
  frozen award exists). The UI keeps the user on the correct step instead of erroring out. This is
  the frontend half of the governed-gate contract (freeze/finalize).

**`ApiErrorCode` type (exported, lines 19ÔÇô24):** union
`"gate_required" | "validation_error" | "not_found" | "unauthenticated" | string`. The four named
codes are the structured envelope's machine codes the UI branches on; the trailing `| string`
keeps it open so unknown codes still type-check (and surface their `detail`).

**`extractError(res)` (internal async, lines 61ÔÇô86):** best-effort message+code extraction from a
non-2xx body. Order: (1) `res.clone().json()`; (2) if object, read `code` (string|null) and `detail`;
(3) if `detail` is a string ÔåÆ `{detail, code}`; (4) if `detail` is a **non-empty array** (FastAPI
validation errors `[{msg, loc, ÔÇĞ}]`) ÔåÆ first element's `.msg`; (5) else fall back to envelope
`title`; (6) else `JSON.stringify(detail)`; (7) on any throw / non-JSON ÔåÆ `{detail: statusText ||
"HTTP {status}", code: null}`. **WHY:** the backend has two distinct error shapes (the structured
`{code,title,detail}` envelope and FastAPI's raw `{detail:[ÔÇĞ]}`); this collapses both into a single
human string + machine code so the UI has one error contract.

**`throwApiError(res)` (internal async, lines 88ÔÇô91):** `extractError` ÔåÆ `throw new ApiError(status, detail, code)`. Return type `Promise<never>`.

**`apiFetch<T>(path, options)` (exported async, lines 99ÔÇô136):**
- inputs: `path: string` (relative to `/api/v1`), `options: {method="GET", body?, signal?}`.
- builds `fetch(`${API_BASE_URL}${API_PREFIX}${path}`, {...})` with: `method`; **`credentials:"include"`**
  (cookie rides every request); `headers` = `{"Content-Type":"application/json"}` **only when a body
  is present** (else `undefined`); `body` = `JSON.stringify(body)` when present; `signal`;
  **`cache:"no-store"`** (never serve a stale RFP figure from cache ÔÇö data fidelity).
- **error handling:** the `fetch` is wrapped in try/catch ÔÇö an `AbortError` (`DOMException`,
  `name==="AbortError"`) is **re-thrown unchanged** (caller-driven cancellation, not an error to
  display); any other thrown error (network down / CORS / backend unreachable) becomes
  `new ApiError(0, message)` ÔÇö **status 0** is the sentinel for "never reached the server".
- on `!res.ok` ÔåÆ `await throwApiError(res)` (typed throw).
- success parsing: `204` ÔåÆ `undefined as T` (e.g. logout); empty body text ÔåÆ `undefined as T`;
  else `JSON.parse(text) as T`.
- **WHY status-0 sentinel:** distinguishes transport failure from an HTTP error so the UI can say
  "backend unreachable" vs surface a server message.

**`apiUpload<T>(path, file, options)` (exported async, lines 147ÔÇô188):** multipart upload.
- builds `FormData`, appends `file` under the part name **`"file"`** with `file.name`; appends any
  extra `fields` (`Record<string, string|number|boolean>`) **stringified** via `String(value)`.
- `fetch` with `method:"POST"`, `credentials:"include"`, **no `Content-Type` header** (the browser
  sets `multipart/form-data` + boundary itself ÔÇö setting it manually would break the boundary),
  `body: form` (**not** `JSON.stringify`'d), `signal`, `cache:"no-store"`.
- same AbortError re-throw / status-0 / `throwApiError` / 204 / empty-text logic as `apiFetch`.
- **WHY separate from apiFetch:** uploads must NOT JSON-encode and must NOT set Content-Type;
  conflating them would corrupt every file ingest (setup workbook, bid import).

**`filenameFromDisposition(header)` (internal, lines 191ÔÇô204):** parses a `Content-Disposition`
header for a filename. Tries RFC-5987 form first (`filename*=UTF-8''name.xlsx`, URL-decoded with a
try/catch fallback to the raw match), then plain `filename=ÔÇĞ`; returns the name or `null`.

**`apiDownload(path, fallbackName)` (exported async, lines 211ÔÇô251):** authenticated binary download.
- `fetch` GET with `credentials:"include"`, `cache:"no-store"`; same AbortError/status-0 handling.
- on `!res.ok` ÔåÆ `throwApiError` ÔÇö **critical WHY:** it does NOT navigate the browser to the URL
  (which would lose the cookie / surface a raw error page); it fetches, and on error throws the
  same typed `ApiError` so download failures get the same inline treatment as every other call.
- success: `res.blob()`; filename = `filenameFromDisposition(content-disposition) ?? fallbackName`;
  creates an object-URL, a temporary `<a download>` anchor, clicks it, removes it; revokes the
  object-URL on the **next tick** (`setTimeout(ÔÇĞ, 0)`) so the click has time to start.
- **WHY:** the no-file-storage contract (ADR-0018) means deliverables (workbooks, archives,
  notices) render on demand and stream to the browser; this is the client side of that stream.
- **side effects:** mutates the DOM (transient anchor), triggers a browser download, allocates +
  revokes an object URL. Returns `Promise<void>`.

**`RequestOptions` (internal interface):** `{method?, body?: unknown, signal?: AbortSignal}`.
**`UploadOptions` (internal interface):** `{fields?: Record<string, string|number|boolean>, signal?}`.

**Exports from this file:** `ApiError` (class), `ApiErrorCode` (type), `apiFetch`, `apiUpload`,
`apiDownload`. Consumed by every domain module + re-exported by `index.ts`.

## C. `lib/format.ts` ÔÇö display formatters (THE decimal display-shape layer)
- **path:** `/home/user/KR_RFP/frontend/lib/format.ts` ┬À **ext** `.ts` ┬À **empty?** no ┬À **lines:** 68 ┬À **census:** row 323 (2096 bytes)
- **what:** Six pure display formatters shared across the UI. **DETAILED WHY:** This is the file
  the audit standard calls out ÔÇö "where decimals get their display shape." The wire carries raw
  numbers (dollars as numbers, percentages as **fractions**); these functions own the
  precision/locale presentation so it lives in exactly one place. `Intl`/`toLocaleString` with
  `undefined` locale = the **browser's locale** (grouping separators, currency placement). Each
  null-coalesces to an **em-dash `ÔÇö`** so a missing value renders as a placeholder, never `NaN`/
  `null`/`0` (which would misstate an RFP figure).

**Every helper, with its FORMATTING + PRECISION rule:**
- `formatBytes(bytes: number): string` ÔÇö human byte size. Guards `!isFinite || <0` ÔåÆ `ÔÇö`. `<1024`
  ÔåÆ `"{bytes} B"` (0 decimals). Else divides by 1024 stepping through `["KB","MB","GB","TB"]`
  (caps at TB). **Precision:** `value.toFixed(value>=10 || unit===0 ? 0 : 1)` ÔåÆ 0 decimals when the
  mantissa ÔëÑ10 or it's the first KB step, else **1 decimal**. e.g. 1536 ÔåÆ "1.5 KB".
- `formatTimestamp(iso: string): string` ÔÇö `new Date(iso)`; on `NaN` time ÔåÆ returns the **raw iso**
  (never throws). Else `toLocaleString(undefined, {year:"numeric", month:"short", day:"numeric",
  hour:"2-digit", minute:"2-digit"})` ÔåÆ e.g. "Jun 22, 2026, 02:30 PM" (locale-dependent). **No
  seconds.**
- `formatPrice(value: number|null): string` ÔÇö null ÔåÆ `ÔÇö`. Else `toLocaleString(undefined,
  {minimumFractionDigits:2, maximumFractionDigits:2})`. **Precision: exactly 2 decimals, no currency
  symbol** (grouping per locale). Used for $/case prices where the `$` is shown separately or
  implied.
- `formatCount(value: number|null): string` ÔÇö null ÔåÆ `ÔÇö`. Else `toLocaleString()` (locale grouping,
  **integer, no forced decimals**).
- `formatMoney(value: number|null, opts?: {cents?: boolean}): string` ÔÇö null ÔåÆ `ÔÇö`. Else
  `toLocaleString(undefined, {style:"currency", currency:"USD", min/maxFractionDigits: cents?2:0})`.
  **Precision: USD currency, WHOLE dollars by default** (0 decimals ÔÇö spend/savings totals are
  large, so cents are noise), **2 decimals when `{cents:true}`**. e.g. 1234567.5 ÔåÆ "$1,234,568"
  (rounds half-up via Intl). **WHY default-whole:** large landed-spend/savings figures read cleaner
  as whole dollars; cents reserved for per-unit contexts.
- `formatPercent(frac: number|null): string` ÔÇö null ÔåÆ `ÔÇö`. Else `toLocaleString(undefined,
  {style:"percent", min/maxFractionDigits:1})`. **Precision: the input is a FRACTION** (`style:
  "percent"` multiplies by 100), rendered to **exactly 1 decimal** with a `%`. e.g. 0.0524 ÔåÆ
  "5.2%". **This is the decimal's last hop** for every `*_pct` field on the wire (which are all
  fractions per types.ts) ÔÇö the ├ù100 + 1-decimal shaping happens here, nowhere else.
- **Note:** no `tabular-nums` / alignment is applied in this file ÔÇö that is a CSS/className concern
  (Layer-3), not the numeric formatting. All six are pure (no side-effects), return `string`.

## F. Layer-1 contributions from this slice (value/precision hops)
The **only numeric transformations** in this slice are the **display-shape hops in `format.ts`**
(Section C) ÔÇö the rest of the client passes values through untouched (`apiFetch` just
`JSON.parse`s). For Layer-1 synthesis, the wireÔåÆpixel hops owned here are:
- **fraction ÔåÆ percent:** `formatPercent(frac)` ÔçÆ `frac ├ù 100`, 1-decimal, `%` suffix
  (`format.ts:61ÔÇô68`). Inputs: every `*_pct` field (all fractions per `types.ts`).
- **dollars ÔåÆ currency string:** `formatMoney(value, {cents?})` ÔçÆ USD, 0 decimals default / 2 with
  `cents`, locale grouping, Intl half-up rounding (`format.ts:47ÔÇô58`).
- **$/case ÔåÆ 2-decimal string:** `formatPrice` ÔçÆ exactly 2 decimals, no symbol (`format.ts:31ÔÇô37`).
- **integer count ÔåÆ grouped string:** `formatCount` (`format.ts:40ÔÇô43`).
- **bytes ÔåÆ human size:** `formatBytes` ÔçÆ /1024 stepping, 0-or-1 decimal rule (`format.ts:4ÔÇô15`).
- **null in any of the above ÔåÆ `ÔÇö`** (em-dash) ÔÇö the universal "missing, not zero" rule.
No FOB+freightÔåÆlanded, unit/pack, renormalization, or force-positive coercion happens in this slice
(those live in the backend engine/ingest); the client only **displays** already-computed numbers.
